Git & Version Control

Mastering Git Commands & Best Practices

Git is a distributed version control system used for tracking changes in source code during software development. It allows multiple people to collaborate, manage project versions, and provides a reliable mechanism to revert back to earlier stages when necessary.

Core Concepts:

  • Repository (repo): A storage location for your project files and history of their changes.
  • Commit: A snapshot of your project at a specific point in time.
  • Branch: A separate line of development, allowing experimentation without affecting the main codebase.
  • Merge: Combining changes from one branch into another.
  • Rebase: Reapply changes from one branch onto another to create a linear history.

Basic Git Commands (Every Developer Must Know):

  • git init - Initialize a new Git repository in the current directory.
  • git clone <url> - Clone a remote repository from platforms like GitHub or GitLab.
  • git add <file> - Stage a file (or use git add . to stage all changes).
  • git commit -m "message" - Commit your changes with a descriptive message.
  • git pull - Fetch and merge changes from a remote repository to your local repo.
  • git push - Push your local changes to a remote repository.

Branching & Collaboration:

  • git branch - List all branches in the repository.
  • git checkout -b <branch> - Create and switch to a new branch.
  • git merge <branch> - Merge changes from another branch into your current branch.
  • git rebase <branch> - Reapply commits from one branch on top of another to maintain a clean commit history.
  • git stash - Temporarily save uncommitted changes, allowing you to work on a clean state without committing.

Advanced Git Commands & Best Practices:

  • git log - View the commit history of your project.
  • git reset --hard <commit> - Reset the current branch to a specific commit, discarding all changes after it.
  • git cherry-pick <commit> - Apply a specific commit from one branch to another.
  • git reflog - View the reference logs, which keep track of all updates made in the repository, even after a branch reset.
  • git tag - Tag a specific commit to create a version reference (useful for marking releases).

Practice Sets:

  • Basic: Set up a repository, create a branch, and commit changes.
  • Intermediate: Work with multiple branches, resolve merge conflicts, and rebase branches.
  • Advanced: Perform cherry-picks, use tags for versioning, and recover from mistakes using git reflog and git reset.

Best Practices for Git:

  • Commit Often: Keep your commits small and focused, making it easier to track changes.
  • Write Clear Messages: Descriptive commit messages help others (and your future self) understand what changed and why.
  • Use Branches: Isolate new features or bug fixes in their own branches to prevent affecting the main codebase.
  • Pull Before Push: Always pull updates from the remote repo before pushing your changes to avoid conflicts.
  • Merge Regularly: Regularly merge your branch with the main branch to stay updated and reduce merge conflicts.
